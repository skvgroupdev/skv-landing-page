# SKV Landing Page Codex Notes

Last scanned: 2026-05-17

## Project Summary

`skv-landing-page` is a Next.js App Router landing site for SKV Group. The main site is a Lao-first marketing page with an English/Lao language toggle, plus a dedicated Lao POS product page and a team profile page.

The app uses:

- Next.js `16.1.1`
- React `19.2.3`
- TypeScript with `strict: true`
- Tailwind CSS v4 through `@tailwindcss/postcss`
- `lucide-react` icons
- `next/image` for local public assets
- Vercel Analytics

## Commands

Run from `skv-landing-page`:

```bash
npm run dev
npm run lint
npm run build
npm run start
```

There are both `package-lock.json` and `pnpm-lock.yaml` files, but `package.json` scripts work with npm. Prefer npm unless the team chooses to standardize on pnpm.

## Important Routes

- `/` is the main landing page in `app/page.tsx`.
- `/pos` is the POS product page in `app/pos/page.tsx`.
- `/team/vongchln` is the Vongvilay profile page in `app/team/vongchln/page.tsx`.
- `/sitemap.xml` is generated by `app/sitemap.ts`.
- `/robots.txt` is generated by `app/robots.ts`.

## Main Files

- `lib/content.ts` stores bilingual English/Lao landing page copy and service data. Icons are imported here and passed through content objects.
- `app/layout.tsx` owns global metadata, icons, Open Graph/Twitter data, and Vercel Analytics.
- `app/globals.css` defines Tailwind v4 imports, CSS variables, Lao/Inter fonts, glass utilities, grid/dot background helpers, and custom animations.
- `components/Navbar.tsx` handles fixed navigation, mobile menu, and language toggle.
- `components/Hero.tsx` renders the main landing hero using `/backgrounds/bg1.webp` and `public/images/vong1.jpg`.
- `components/Services.tsx` renders the three service cards from `t.services.items`.
- `components/Awards.tsx` renders the Muan Awards/portfolio section.
- `components/MyCustomer.tsx` renders trusted customer cards.
- `components/Partners.tsx` renders partner/customer logos.
- `components/Footer.tsx` renders contact details.
- `components/SystemPreviewTabs.tsx` is the interactive POS screen mockup used on `/pos`.

## Content And Language Notes

- The home page starts in Lao: `useState<Language>("la")`.
- The home page can toggle to English using `content[lang]`.
- The POS page is intentionally hardcoded to Lao copy right now.
- Several visible headings are hardcoded Lao inside components instead of coming from `lib/content.ts`.
- Some sections are currently commented out on the home page: `About`, `Contact`, and `Team`.
- Navbar links include anchors such as `#about` and `#contact`; `#contact` exists on `Footer`, but `#about` is currently not rendered because `About` is commented out.

## Styling Notes

- The landing page mixes a bright white content area with a blue hero.
- The POS page uses a dark premium theme with amber and blue accents.
- Global CSS defines dark theme variables, but the home page overrides with `bg-white text-slate-900`.
- Custom animations used in markup include `animate-float`, `animate-fade-in-up`, and `animate-pulse-glow`.
- There are references to animation classes such as `animate-blob` and `animate-spin-slow` that are not defined in `app/globals.css` from this scan.

## Development Priorities

Recommended next steps:

1. Clean lint errors while keeping behavior unchanged.
2. Add reusable TypeScript types for content props instead of `any`.
3. Remove unused imports and unused variables.
4. Decide whether `About`, `Team`, and `Contact` should be active sections or remove/update their nav links.
5. Standardize package manager lockfiles.
6. Check undefined animation classes and either define or remove them.
7. Consider moving hardcoded section copy into `lib/content.ts` for easier Lao/English maintenance.

## Working Conventions

- Keep Lao text intact unless the requested task is copy editing or translation.
- Prefer editing centralized content in `lib/content.ts` when changing services, hero text, team data, and reusable labels.
- Use local `public` images through `next/image`.
- Preserve the App Router structure under `app/`.
- Use existing Tailwind utility style rather than introducing a separate component library.
- After UI edits, run `npm run lint`; for bigger changes, run `npm run build`.
